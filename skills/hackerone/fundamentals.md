# HackerOne API: Fundamentals

Based on HackerOne API v1 documentation.

## Overview

The HackerOne API is a REST API returning JSON responses compliant with the **JSON:API specification**. All access is exclusively through HTTPS at `https://api.hackerone.com/`. There are two API surfaces:

- **Customer API** — for program managers (`/v1/{resource}`)
- **Hacker API** — for security researchers (`/v1/hackers/{resource}`)

## Authentication

HTTP Basic authentication. Credentials = API token identifier (username) + token value (password).

```bash
# Customer API
curl "https://api.hackerone.com/v1/reports/129329" \
  -u "<API_IDENTIFIER>:<API_TOKEN>"

# Hacker API
curl "https://api.hackerone.com/v1/hackers/reports/129329" \
  -u "<USERNAME>:<API_TOKEN>"
```

**Customer tokens:** Generated by admin users from organization settings.
**Hacker tokens:** Generated from user Settings page.

### Auth Errors

| Code | Meaning |
|------|---------|
| 401 | Invalid credentials or missing identification |
| 403 | Token lacks permissions or resource belongs to another program |

## Rate Limits

| Operation | Limit |
|-----------|-------|
| Read (general) | 600 requests/minute |
| Read (report pages) | 300 requests/minute |
| Write | 25 requests/20 seconds |

Exceeding limits returns **HTTP 429 Too Many Requests**.

## Versioning

URL includes global version: `https://api.hackerone.com/{version}/{resource}`

- No default version — requests **must** specify version (currently `v1`)
- Adding new attributes or resources is not a breaking change

## Error Responses

| Code | Status | Details |
|------|--------|---------|
| 400 | Bad Request | Non-conforming request |
| 404 | Not Found | Resource unavailable |
| 406 | Not Acceptable | Unsupported response format |
| 422 | Unprocessable Entity | Correct syntax but can't process |
| 429 | Too Many Requests | Rate limit exceeded |
| 500 | Internal Server Error | Server-side issue |
| 503 | Service Unavailable | Servers offline |

Error response format:
```json
{
  "errors": [
    { "status": 404, "detail": "Resource not found" }
  ]
}
```

## JSON:API Format

All responses follow JSON:API. Key structure:

```json
{
  "data": {
    "id": "12345",
    "type": "report",
    "attributes": { ... },
    "relationships": {
      "reporter": { "data": { "id": "1", "type": "user" } }
    }
  }
}
```

- **Single resource:** `data` is an object
- **Collection:** `data` is an array
- **Relationships:** nested `data` with `id` and `type` references
- **Timestamps:** ISO 8601 with millisecond precision

## Pagination

Collections support cursor-based pagination:

| Parameter | Default | Max |
|-----------|---------|-----|
| `page[number]` | 1 | — |
| `page[size]` | 25 | 100 |

Response includes `links` object:
```json
{
  "links": {
    "self": "https://api.hackerone.com/v1/...",
    "next": "https://api.hackerone.com/v1/...?page[number]=2",
    "last": "https://api.hackerone.com/v1/...?page[number]=10"
  }
}
```

For datasets > 10,000 items, use ID-based filtering (`filter[id__gt]`) instead of deep pagination.

## Common Patterns

### Filtering Reports

```bash
# Filter by program
curl "https://api.hackerone.com/v1/reports?filter[program][]=myprogram" \
  -u "$API_ID:$API_TOKEN"

# Filter by state
curl "https://api.hackerone.com/v1/reports?filter[state][]=triaged&filter[state][]=retesting" \
  -u "$API_ID:$API_TOKEN"

# Filter by custom field
curl "https://api.hackerone.com/v1/reports?filter[custom_fields][CF_ATTRIBUTE_ID]=value" \
  -u "$API_ID:$API_TOKEN"
```

### Incremental Sync

Use `updated_at_after` parameter to fetch only changes since last sync:

```bash
curl "https://api.hackerone.com/v1/incremental/activities?handle=myprogram&updated_at_after=2024-01-01T00:00:00Z" \
  -u "$API_ID:$API_TOKEN"
```

The response includes `meta.max_updated_at` — save this for next sync.

### Content-Type Headers

- Most requests: `Content-Type: application/json`
- File uploads: `Content-Type: multipart/form-data`

## URL Structure Quick Reference

| API | Pattern | Example |
|-----|---------|---------|
| Customer | `/v1/{resource}` | `/v1/reports`, `/v1/programs` |
| Hacker | `/v1/hackers/{resource}` | `/v1/hackers/reports`, `/v1/hackers/programs` |
| Organizations | `/v1/organizations/{org_id}/{resource}` | `/v1/organizations/123/members` |
| Program-scoped | `/v1/programs/{id}/{resource}` | `/v1/programs/456/structured_scopes` |
| Report actions | `/v1/reports/{id}/{action}` | `/v1/reports/789/bounties` |
